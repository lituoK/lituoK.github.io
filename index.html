但是这样做会TLE，所以我们就要通过一些方法优化一下时间复杂度。我们可以想象一个容器，我们可以把数插入进去，然后在很短的时间内，查询到这个容器中有多少个数是在给定范围的。如果存在这样一个数据结构，我们就可以很方便的遍历前缀和数组sum,然后查询容器中有多少数是在范围[sum[i] - upper, sum[i] - lower]中的，然后我们再将sum[i]插入到容器中去。

思路

统计每个数对应的二进制数的1的个数，然后根据1的个数排序即可。

怎么统计二进制数中1的个数呢？我们可以遍历这个数的每一位，如果这位是1，就计数。这样就可以得到结果了。

这样每个数都要遍历32位，在python中一个数可能也不止32位，我们如何得到这个数的1的个数呢？

可以使用lowerbit算法，lowerbit是一个常用的算法，可以用来得到一个数的最低一位1所在的位置，比如说 $ 6d = 110b $ ，6的最低一位1出现在第二位，lowerbit算法就会返回 $ 10b = 2d $ (b和d分别表示二进制和十进制)。同理 $ lowerbit(8) = 8, lowerbit(12) = 4, …. $

lowerbit算法的代码很简单，只有一行具体的证明请同学们自己思考或者搜索。

那么我们可不可以继续优化这个算法的时间复杂度呢？我们可以通过动态规划来进行优化, 给定一个数x，如果我们已知 $ x - lowerbit(x) $ 中有多少个1，那么x就比 $ x - lowerbit(x) $ 多一个1.状态转移方程：

f(x) = f(x - lowerbit(x)) + 1

57. 插入区间

发表于 2020-11-04

题目

57. 插入区间

思路

这道题就是一个区间合并的题，首先给定一组有序的区间，和一个要插入的区间，我们可以把这组区间分成三类

  1. 在要插入的区间左边，没有交集的区间
  2. 和给定区间可以合并成一个区间的区间
  3. 最后是剩下的和合并后的区间没有交集的区间。

设：给定区间的左右断点分别为l和r。

对于三类区间：

  • 首先，因为给定的区间都是有序的，所以我们可以先把这些数组加入到答案中，这些数组的都在给定区间的左边，所以他们的右端点是一定在给定区间的左端点的左边的，也就是$intervals[i][1] < l$。

  • 对于第二类区间，是我们要处理的区间，因为我们要将他们进行合并，先想一想他们具有什么样的特征，因为我们目前已经把$intervals[i][1] < l$的区间都排除掉了，所以剩下的区间一定满足$intervals[i][1] >= l$；然后在这些区间里面，什么样的区间是可以和我们给定的区间进行合并的呢？我们先将区间[l, r]初始化为给定的区间，然后我们将所有可以合并的区间合并进去。如果$intervals[i][0] <= r$那么就表示，这个区间的左端点是在我们给定的区间内的，我们就更新区间[l, r]

  • 对于第三类区间，和第一类相同，直接加入到答案里面。

349. 两个数组的交集

发表于 2020-11-02

题目

349. 两个数组的交集

思路

使用一个hash表，存储一个数组的所有元素，然后再遍历另一个数组，如果hash表中存在这个元素，就把它加入到答案中。

因为答案是没有重复的，所以在遍历另一个数组的时候，在向答案数组中添加元素的时候，我们就把这个元素从hash表中删除，这样就不会有重复的元素了。
